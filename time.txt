Описание задачи
Разработать парсер задач с сайта codefeorces, который будет собирать данные о задачах и их свойствах,
сохранять их в базу данных и обновлять информацию каждый час. Так же необходимо создать Telegram-бота 
для взаимодействия с пользователями, предоставляющего возможность поиска и подбора задач по сложности и темам.

Требования
1. Собрать данные о задачах с сайта Codeforces, включая:
-Темы (например, "математика", "перебор", "графы").
-Количество решений задач.
-Название и номер задачи.
-Сложность задачи (например, 800б 900 и т.д.)
2. Сохранять и обновлять данные в базе данных, добавляя нове задачи при необходимости.
3. Настроить парсинг страниц Codeforces с периодичностью 1 час.
4. Разработать Telegram-бота, который позволи:
-Выбирать сложность и тему задач, с выводом соответствующей подборки.
-Искать задачи по запросу, с выводом всей информации о найденной задаче.
5. Алгоритм подбора задач:
-Для выбранной сложности и тематики получать подборку из 10 задач.
-Обеспечить, чтобы задачи не пересекались между различными контестами, т.е. каждая задача принадлежала только одному контесту.

Технические требования:
Язык программирования: Python3.11+
СУБД: PostgreSQL
Код размещен в открытом репозитории (Git).
Код покрыт тестами с покрытием не менее 75%.
Доступна документация (Readme файл).
Код оформлен согласно PEP8.
Использование ORM SQLAlchemy для работы с базой данных.
Реализация проекта с использованием Docker и Docker-compose.
Интеграция с Telegram.
Решение должно быть оптимальным с точки зрения алгоритмической сложности.
Проект должен быть реализован с использованием ООП подхода.

Декомпозиция задачи:
1 часть:
	создать класс для получения данных в фотмате json с сайта codeforces
	создать  функцию обработчик данных json 
	создать функцию для заполнения БД обработанными данными.
2. 
	Создать сеодинение с БД Postgres с помощью SQLAlchemy и создать 2 таблицы (описание задач и статистика задач) со следующими данными:
	- id, номер задачи(состоит из номера контеста и литеры), название задачи, сложность задачи, тема(ы) задачи
	- id, номер задачи(состоит из номера контеста и литеры), количество решений задачи
	
	
3.
	Создать бесконечный цикл в котором сохранять время начала и оновлятб его каждый час (этот бесконечный цикл должен как считать время, так и 
		принимать команды)
	Каждый час брать данные по задачам codeforces и сравнивать размер полученных данных с размерностью БД: если размер файла больше - то 
		добавить данные в БД
	

4.	
	Создать класс TG_bot,
	Создать функции обработки сообщений или кнопок с ТГ бота,
	Создать функцию с выборкой данных из БД
	Создать функцию отправки сообщения в ТГ бота
5.
	
	Заджоинить обе таблицы по номеру и литере, отобрать задачи по указанной сложности,
	Задача в выборке принадлежит только одному контесту(вариант 1: делать выборку из результативной таблицы и вставлять результат в список
	предварительно придется сделать список экземпляров класса для последующей отправки в бота
	вариант 2: можно сделать промежутоную таблицу, в которую перед вставкой строки проверять существуетли строка с таким же номером контеста)

в результате в главной функции должно быть следующее:
при запуске программы - должна производиться проверка на существование БД, если БД есть, то запрос к пользоваетлю, нужно ли переписывать БД
если БД нетт, то создаем новую и запоминаем время, от этого времени будем каждый час обновлять БД.
Бесконечный цикл должен постоянно слушать
